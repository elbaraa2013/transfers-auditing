# مواصفات تطبيق الموبايل — نظام تدقيق الحوالات

> هذا مستند مواصفات تفصيلي لبناء تطبيق موبايل (Android و iOS) مطابق لنظام تدقيق الحوالات الحالي (نسخة الويب). انسخ هذا المستند كاملاً والصقه لأي أداة ذكاء اصطناعي لتنفيذ التطبيق.

---

## 1. نظرة عامة

تطبيق محاسبي لتدقيق حوالات المندوبين (مندوبي المبيعات). صاحب العمل يستلم من عملائه إيصالات تحويل بنكي (تطبيق بنكك) أو دفعات نقدية عن طريق مندوبين، ويحتاج إلى:

- تسجيل كل حوالة (بالمسح الضوئي لصورة الإيصال أو يدوياً).
- التحقق من صحة الحوالة واعتمادها أو رفضها (التدقيق/المطابقة).
- متابعة حساب كل مندوب (كشف حساب) وأرصدة الحوالات المعتمدة.
- ملخصات وإجماليات لكل المندوبين خلال فترة محددة.

**اللغة:** عربي فقط، الاتجاه من اليمين لليسار (RTL) في كل الشاشات.

**الهوية البصرية:** ألوان بيج/ذهبي/أسود:
- خلفية فاتحة بيج: `#FAF4E3`
- ذهبي أساسي: `#A6791E` وذهبي غامق: `#8A6718`
- بيج حدود: `#E8D9A8` و `#EAD9B0`
- أسود/بني غامق للرأس والبطاقات الداكنة: `#1C1A17`
- شعار المؤسسة (الوابل) يظهر في شاشة الدخول وأعلى التطبيق.

**العملة:** جنيه سوداني، تُعرض هكذا: `50,000 ج.س` (أرقام إنجليزية بفواصل الآلاف).

**التقنية المقترحة:** React Native (Expo) أو Flutter — مع دعم كامل للـ RTL والخط العربي.

---

## 2. المستخدمون والدخول

- التطبيق متعدد المستخدمين: كل مستخدم (صاحب عمل) يرى بياناته فقط (عزل كامل للبيانات بين الحسابات).
- تسجيل الدخول بالبريد الإلكتروني وكلمة المرور (إنشاء حساب + تسجيل دخول + تسجيل خروج).
- كل البيانات (مندوبين، حوالات) مرتبطة بمعرف المالك `ownerId`، وكل استعلام يجب أن يُفلتر به.

## 3. نموذج البيانات

### المندوب (Agent)
| الحقل | النوع | ملاحظات |
|---|---|---|
| id | رقم تسلسلي | |
| ownerId | نص | مالك الحساب |
| name | نص | اسم المندوب |
| phone | نص | رقم الهاتف |
| lastActivityAt | تاريخ/وقت | آخر نشاط (آخر حوالة سُجلت له) |

**الرصيد لا يُخزَّن** — يُحسب دائماً كمجموع مبالغ الحوالات **المعتمدة** للمندوب.

### الحوالة (Transfer)
| الحقل | النوع | ملاحظات |
|---|---|---|
| id | رقم تسلسلي | |
| ownerId | نص | مالك الحساب |
| operationNumber | نص | رقم العملية من الإيصال — **إلزامي** |
| amount | رقم | المبلغ — **إلزامي** |
| fromAccount | نص اختياري | حساب المرسل (16 رقم تقريباً) |
| toAccount | نص اختياري | حساب المستلم |
| recipientName | نص اختياري | اسم المرسل إليه |
| comment | نص اختياري | تعليق |
| status | pending / approved / rejected | تبدأ pending |
| rejectionReason | نص اختياري | سبب الرفض |
| riskLevel / riskScore | مستوى ودرجة مخاطرة | منخفض/متوسط/مرتفع |
| paymentMethod | bankak / cash | تحويل بنكي أو نقدي |
| agentId | رقم | المندوب |
| transferDate | نص (YYYY-MM-DD) اختياري | **تاريخ الحوالة الفعلي** (من الإيصال أو إدخال يدوي) |
| createdAt | تاريخ/وقت | تاريخ التسجيل في النظام |

### قواعد عمل أساسية (مهم جداً)
1. **رقم العملية فريد لكل مستخدم** لكن **الحوالات المرفوضة لا تحجز الرقم**: يجوز إعادة تسجيل رقم عملية سبق رفضه، ولا يجوز تكرار رقم لحوالة معلقة أو معتمدة. عند محاولة تسجيل رقم مكرر تظهر رسالة: «رقم العملية مسجل بالفعل».
2. **تاريخ الحوالة المعتمد في كل العرض والفلترة والتقارير هو `transferDate`** (تاريخ الحوالة الفعلي)، وإذا كان فارغاً نرجع لـ `createdAt`. المقارنة بصيغة `YYYY-MM-DD`.
3. الحذف مسموح للحوالات **المعلقة والمرفوضة فقط** — الحوالة المعتمدة لا تُحذف ولا تُعدَّل (مع رسالة تأكيد قبل الحذف).
4. الاعتماد/الرفض لا يتم إلا على حوالة معلقة (عمليات ذرّية لمنع التعارض).
5. درجة المخاطرة تعتمد على حجم المبلغ (كلما زاد المبلغ ارتفعت الدرجة) وتنقص اكتمال البيانات.

## 4. شاشات التطبيق

قائمة تنقل سفلية (Bottom Tabs) أو قائمة جانبية RTL بالأقسام التالية:

### 4.1 لوحة النظرة العامة (الرئيسية)
- 4 بطاقات إحصائية: إجمالي الحوالات، المعلقة، المعتمدة، المرفوضة (العدد + المبلغ).
- جدول/قائمة «أحدث الحوالات» (رقم العملية، التاريخ، المندوب، المبلغ، الحالة).

### 4.2 مسح إيصال (المسح الضوئي) — الميزة الأهم
- المستخدم يختار المندوب أولاً، ثم يلتقط صورة الإيصال بالكاميرا أو يختار عدة صور من المعرض.
- تُرسل كل صورة (base64) إلى الخادم `POST /api/scan` الذي يستخرج بالـ OCR: رقم العملية، المبلغ، من حساب، إلى حساب، اسم المرسل إليه، التاريخ، ودرجة المخاطرة والثقة.
- تظهر بطاقة نتيجة لكل صورة، مع إمكانية **تعديل يدوي** للحقول: المرسل إليه، من حساب، إلى حساب، **والتاريخ (منتقي تاريخ)** — لأن الـ OCR قد يخطئ أو يعجز عن القراءة.
- الحقول الإلزامية للتسجيل: رقم العملية والمبلغ. إذا نقصت تظهر رسالة «تعذّر قراءة حقول إلزامية… استخدم صورة أوضح».
- زر «تسجيل» لكل بطاقة وزر «تسجيل الكل»، وتُسجَّل الحوالة بحالة «معلقة».

### 4.3 دفع نقدي
- نموذج بسيط لتسجيل دفعة نقدية: المندوب، المبلغ، تاريخ الدفع (افتراضي اليوم وقابل للتعديل)، اسم الدافع (اختياري)، تعليق (اختياري).
- تُسجَّل كحوالة بطريقة دفع «نقدي» وتأخذ رقم عملية تلقائي.

### 4.4 مطابقة الحوالات (التدقيق)
- بطاقات للحوالات **المعلقة فقط**: رقم العملية، **تاريخ الحوالة**، المبلغ كبير وواضح، المندوب، المرسل إليه، شارة درجة المخاطرة، وصورة الإيصال إن وجدت.
- أزرار: **اعتماد** ✓، **رفض** ✗ (مع إدخال سبب الرفض)، **حذف** (مع تأكيد).

### 4.5 إدارة الحوالات
- جدول/قائمة بكل الحوالات مع بحث (برقم العملية/المندوب/المرسل إليه) وفلترة بالحالة.
- الأعمدة: رقم العملية، **تاريخ الحوالة**، المندوب، المرسل إليه، المبلغ، المخاطرة، الحالة.
- حذف الحوالات المعلقة/المرفوضة فقط (المعتمدة مقفولة).

### 4.6 كشف حساب المندوب
- اختيار مندوب، إضافة مندوب جديد (اسم + هاتف)، تعديل بيانات مندوب.
- رأس داكن: اسم المندوب وهاتفه + **الرصيد الحالي (المعتمد)**.
- **فلتر فترة (من تاريخ / إلى تاريخ):** عند تحديد فترة، كل الأرقام تعكس الفترة فقط:
  - الرأس يتحول إلى «إجمالي المعتمد (الفترة)».
  - 4 بطاقات: إجمالي الحوالات، المعتمدة، المعلقة، المرفوضة (عدد + مبلغ) — للفترة.
  - جدول حركة الحساب: التاريخ (تاريخ الحوالة)، رقم العملية، المرسل إليه، طريقة الدفع، المبلغ، الحالة.
- **زر طباعة / تصدير PDF / مشاركة** لكشف الحساب (بنفس منطق الفترة).
- قسم «ملخص كل المندوبين»: جدول لكل مندوب (عدد الحوالات، إجمالي المبلغ، المعتمد، المعلق، المرفوض) + صف «الإجمالي العام»، مع فلتر فترة مستقل — **الفلترة بتاريخ الحوالة وليس تاريخ التسجيل**.

### 4.7 المندوبون غير النشطين
- قائمة بالمندوبين الذين لم تُسجَّل لهم حوالة منذ **48 ساعة** أو أكثر، مرتبة بالأطول خمولاً، مع عدد ساعات الخمول.

### 4.8 تذكير واتساب
- لكل مندوب زر يفتح واتساب (`wa.me/رقم-الهاتف`) برسالة تذكير جاهزة قابلة للتعديل.

## 5. الخادم (Backend)

خياران:

**الخيار أ (مفضّل إن أمكن):** ربط التطبيق بالخادم الحالي القائم (REST API بمسارات مثل):
- `POST /api/scan` — مسح صورة إيصال (OCR)
- `GET/POST /api/transfers` + `PATCH /api/transfers/:id/approve|reject` + `DELETE /api/transfers/:id`
- `GET/POST/PATCH /api/agents`
- `GET /api/agents/:id/statement` — كشف حساب
- `GET /api/agents/summary?from&to` — ملخص كل المندوبين لفترة
- `GET /api/agents/inactive`

**الخيار ب:** بناء خادم جديد بنفس نموذج البيانات وقواعد العمل أعلاه (قاعدة بيانات PostgreSQL)، مع خدمة OCR للإيصالات (Tesseract مع دعم العربية والإنجليزية، مع معالجة مسبقة للصور الملونة).

## 6. ملاحظات تنفيذية مهمة

1. كل الواجهات RTL بالكامل، بما فيها الجداول والأيقونات والتنقل.
2. التواريخ تُعرض بأرقام إنجليزية بتنسيق عربي (مثال: «29 يونيو 2026»).
3. المبالغ دائماً بفواصل آلاف + «ج.س».
4. حالات الحوالة بألوان ثابتة: معلقة (كهرماني)، معتمدة (ذهبي/أخضر داكن)، مرفوضة (أحمر).
5. رسائل الخطأ والتأكيد كلها بالعربية (أمثلة: «تم التسجيل»، «رقم العملية مسجل بالفعل»، «هل أنت متأكد من حذف الحوالة؟»).
6. يجب أن يعمل المسح الضوئي من كاميرا الهاتف مباشرة مع ضغط الصورة قبل الرفع.
7. دعم طباعة/مشاركة كشف الحساب كـ PDF من الهاتف.
